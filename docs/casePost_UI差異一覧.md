# 画像のUI と 現状の casePost LWC の差異一覧

## 1. ページ上部（レコードヘッダー）

| 項目 | 画像のUI | 現状のLWC |
|------|----------|-----------|
| レコードタイプ・パンくず | 「お問い合わせ」＋アイコン | **なし**（コンポーネント外のため未実装） |
| レコードタイトル | 「お問い合わせテスト」を大きく表示 | **なし** |
| アクションボタン | 「+ フォローする」 | **なし** |
| キー情報バー | お問い合わせ番号・優先度アイコン・ステータス・所有者を横並び | **なし** |
| ステータスパス | 新規→対応中→…→クローズの進捗バー | **なし** |
| 「ステータスを完了としてマーク」ボタン | あり | **なし** |

※ これらは通常、レコードページの標準ヘッダー側で表示されるため、LWC単体では持っていない。

## 2. 左カラムの構成

| 項目 | 画像のUI | 現状のLWC |
|------|----------|-----------|
| タブ | **詳細**・タイムライン・関連・ナレッジ・Slack | **なし**（タブなし） |
| 「詳細」の扱い | **タブ名**。タブ選択時にメインの項目＋折りたたみセクションを表示 | **アコーディオン1つ目**の見出しとして「詳細」を表示 |
| メイン項目の表示方法 | タブ「詳細」内で、項目を**一覧でそのまま表示**（折りたたみではない） | すべて「詳細」「詳細情報」「システム情報」の**アコーディオン3つ**に分けて表示 |
| 編集アイコン | 編集可能項目に**鉛筆アイコン**あり | **なし** |
| ラベル補助 | 優先度などに**情報アイコン**あり | **なし** |

## 4. 右カラム（メッセージ・投稿）

| 項目 | 画像のUI | 現状のLWC |
|------|----------|-----------|
| タブ | メッセージ・ファイル | 同じ（メッセージ・ファイル） |
| 投稿エリアの**順序** | **上**: 「最新情報を共有...」の入力＋**共有**ボタン<br>**下**: 複数行のテキストエリア | **上**: 「投稿」ラベル＋複数行テキストエリア（投稿内容を入力）<br>**下**: 「最新情報を共有...」入力＋**共有**ボタン |
| 投稿の見た目 | 1行入力＋共有が上、その下に長文用エリア | 長文用エリアが上、その下に1行入力＋共有 |

## 5. レイアウト・見た目

| 項目 | 画像のUI | 現状のLWC |
|------|----------|-----------|
| 左カラム幅 | 詳細タブ＋フィールドが主で、やや広め | 約25%（1/4）のサイドバー |
| 右カラム幅 | メッセージ／ファイルが主 | 約75%（3/4） |
| 左の折りたたみ | 「詳細情報」「システム情報」のみ折りたたみ（下向き chevron） | 「詳細」「詳細情報」「システム情報」の3つがアコーディオン（右向き chevron で開閉） |

---

## まとめ（修正すると近づくポイント）

1. **右カラムの投稿エリアの順序**  
   「最新情報を共有...」＋共有ボタンを**上**、その下に複数行テキストエリアにすると画像に近い。

2. **左カラムの構成**  
   「詳細」をタブではなく「最初から開いた状態の項目一覧」にし、その下に「詳細情報」「システム情報」だけ折りたたみにすると、画像の動きに近い。

3. **編集・情報アイコン**  
   編集可能項目に鉛筆アイコン、必要に応じて情報アイコンを付けると見た目が揃う。

4. **経過時間・解決期限・解決残り時間**  
   カスタム項目またはSLA項目を用意すれば表示可能（標準のみの場合は従来どおり未表示のまま）。

5. **レコードヘッダー・ステータスパス**  
   レコードページの標準レイアウト側で表示する想定なら、LWCでは変更不要。
